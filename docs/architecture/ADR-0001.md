## Context

Project Pulse is a new offline-first Android MVP that needs to display projects, tasks, completion scores, and hours. We needed to choose a stack that is modern, maintainable, and consistent with the Pixel Lab workspace.

## Decision

Adopt the following architecture and stack:

- **Language**: Kotlin
- **UI**: Jetpack Compose with Material 3 and dynamic theming
- **State management**: MVVM with `ViewModel` + Compose `collectAsStateWithLifecycle`
- **Navigation**: Navigation Compose with type-safe destinations
- **Persistence**: Room with coroutines/Flow
- **Dependency injection**: Hilt
- **Build tooling**: Gradle Kotlin DSL, AGP 8.6.0, Gradle 8.7, compileSdk 35, minSdk 26, JVM 17

## Consequences

- Pros: Aligns with workspace conventions in `daily-trendscape`; modern, testable, offline-first; leverages Google-recommended Android architecture.
- Cons: Adds build complexity compared to a simple prototype; Hilt and Room require KSP setup.

## Alternatives Considered

- **Flutter/RN** — Rejected: user requested Android native app; workspace already has Kotlin Android expertise.
- **Manual SQLite/OpenHelper** — Rejected: Room provides compile-time SQL validation, coroutines/Flow integration, and migrations.
- **Koin instead of Hilt** — Rejected: Hilt is already used across Pixel Lab Android projects; keeps DI consistent.
